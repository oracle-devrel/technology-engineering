# Scripts / OpenShift

The directory contains OpenShift-focused manifests, command snippets, and helper scripts used to configure, validate, and operate clusters in a consistent way. Typical content includes post-install checks, Day 2 routines (for example adding nodes, applying MachineConfigs, updating cluster settings), and reusable YAML templates for platform components such as registry, logging, and common operators. The goal is to keep operational steps repeatable and auditable by capturing “known good” commands and configuration artifacts that can be applied across multiple clusters and environments.
