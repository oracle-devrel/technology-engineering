
# Day2-Operationss

Covers everything that happens after the initial OpenShift cluster is up and running on OCI. It documents repeatable operational procedures such as scaling and adding nodes, ISO-based node onboarding, applying MachineConfig changes, enforcing naming and configuration standards, planning and executing upgrades, and performing routine validation. The goal is to keep Day 2 work predictable, auditable, and consistent across environments while reducing risk during changes.
